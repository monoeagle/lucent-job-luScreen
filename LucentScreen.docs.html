Überblick

LucentScreen ist ein leichtgewichtiges Windows-Screenshot-Tool mit Tray-Integration, globalen Hotkeys, Annotations-Editor und Verlauf. Es läuft als WPF-Anwendung in Windows PowerShell 5.1 — kein Compile-Step, kein .NET-Runtime-Install, läuft auf jedem Windows 10/11.

Verteilung erfolgt als signiertes Bundle, das vom Softwareverteilteam in einen MSI-Installer verpackt wird.

Schnellstart

  1. App über die Softwareverteilung installieren lassen (oder portable: ZIP entpacken + LucentScreen.exe starten)
  2. Tray-Icon erscheint in der Taskleiste (rechte untere Ecke, ggf. unter dem ↑-Pfeil)
  3. Strg+Shift+1 drücken → Vollbild-Overlay → Maus-Drag → Region wird als PNG gespeichert + im Clipboard
  4. Im Verlauf (Strg+Shift+H) Doppelklick auf ein Bild öffnet den Editor

Tray-Icon

LucentScreen läuft im Hintergrund. Das Icon liegt im System-Tray (rechte untere Ecke der Taskleiste). Es öffnet sich kein Hauptfenster.

Wenn du das Icon nicht siehst: Klick auf den ↑-Pfeil im Tray-Bereich — Windows blendet selten genutzte Icons standardmäßig aus. Dauerhaft sichtbar machen: Rechtsklick auf die Taskleiste → Taskleisten-Einstellungen → Infobereich → LucentScreen auf Ein.

Hotkeys

LucentScreen registriert globale Tastenkombinationen über die Windows-API RegisterHotKey. Sie funktionieren in jeder Anwendung — auch wenn LucentScreen kein Fokus-Fenster hat.

Belegung ändern: Tray → Konfiguration… → GroupBox „Hotkeys" → in das Feld klicken und gewünschte Kombi drücken. Speichern registriert die Hotkeys live neu — kein App-Neustart nötig. (Die Tray-Menü-Anzeige aktualisiert sich erst beim nächsten App-Start.)

Speichern

Strg+S rendert Bitmap + Annotations bei Zoom 1.0 in PNG-Auflösung, schreibt nach <original>_edited.png, kopiert das Bild ins Clipboard. Editor schließt automatisch + Toast „Gespeichert + kopiert".

ESC bei ungespeicherten Änderungen zeigt einen Confirm-Dialog (Ja/Nein).

Process-Modell

  • Single-Instance via System.Threading.Mutex (Global\LucentScreen.SingleInstance)
  • STA-Apartment zwingend für WPF + Clipboard — Self-Relaunch wenn nicht im STA
  • DPI-Awareness PER_MONITOR_AWARE_V2 als allererster Schritt nach Logging

Stolpersteine

Auswahl der ärgerlichsten WPF-/PowerShell-5.1-Fallen, in die LucentScreen während der Entwicklung gelaufen ist. Ausführlicher Pool projektübergreifend in _Projects/.erkenntnisse/wpf-powershell-gotchas.md.

  • BitmapImage + DecodePixelWidth liefert leere Tiles → Workaround: BitmapFrame.Create + TransformedBitmap
  • FileSystemWatcher mit ScriptBlock-Handler killt den Prozess silent vom Worker-Thread → DispatcherTimer-Polling
  • Modul-private _Underscore-Funktionen sind aus .GetNewClosure()-Closures unauffindbar → Helper als ScriptBlock-Variable
  • $script:Variable = … im Closure schreibt in einen anderen Scope → Hashtable-Slot oder Inplace-Update
  • PS 5.1 liest psm1 ohne BOM als CP-1252 → Umlaute werden Mojibake; mit BOM speichern
  • Verschachtelte @(@(...),@(...)) mit Operatoren werfen op_Addition auf [Object[]] → innere Arrays als Hashtables
  • .GetNewClosure() friert Forward-Refs als $null → Slot via shared $state
  • $rtb.Render($visual) rendert mit Layout-Offset → VisualBrush auf DrawingVisual bei (0,0)

Troubleshooting

Tray-Icon nicht sichtbar: Klick auf den ↑-Pfeil neben der Uhr — Windows blendet selten genutzte Icons standardmäßig aus. Dauerhaft sichtbar: Taskleisten-Einstellungen → Infobereich → LucentScreen auf Ein.
Tray-Icon bleibt nach Beenden sichtbar: Windows-Glitch. Über das Icon mit der Maus fahren, dann verschwindet es.
Hotkey funktioniert nicht: Andere Anwendung hat die Kombination bereits registriert. Im App-Log %LOCALAPPDATA%\LucentScreen\logs\app.log steht die Win32-Fehlernummer. In der Konfig auf alternative Kombi wechseln.
Screenshot auf zweitem Monitor ist verschoben: DPI-Skalierungs-Mismatch — Windows-Display-Einstellungen prüfen, App neu starten. (DPI-Tests im Multi-Monitor-Setup sind AP 10.)
Konfig-Dialog Save scheint nichts zu tun: Inline-Validation-Box am unteren Rand prüfen. MessageBox erscheint zusätzlich bei Validation-Fehler oder Exception (mit Quellzeile).
Editor-Save speichert verschobenes Bild: behoben in v0.2.0 (VisualBrush-Workaround). Falls trotzdem Versatz: bitte Issue mit Screenshot.
App-Log: %LOCALAPPDATA%\LucentScreen\logs\app.log mit 7-Tage-Rotation. Editor-Debug-Log via ./run.ps1 sd + ./run.ps1 lg.
