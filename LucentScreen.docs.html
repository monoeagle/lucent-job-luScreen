Überblick

LucentScreen ist ein leichtgewichtiges Windows-Screenshot-Tool mit Tray-Integration, globalen Hotkeys, Annotations-Editor und Verlauf. Es läuft als WPF-Anwendung in PowerShell 7+.

Verteilung erfolgt als signiertes Bundle, das vom Softwareverteilteam in einen MSI-Installer verpackt wird.

Installation

Wird mit AP 9 (Packaging) gefüllt. Quelle: luscreen-docs/docs/grundlagen/installation.md.

Schnellstart

Wird mit AP 2/3 (Tray + Hotkeys) gefüllt.

Tray-Icon

Wird mit AP 2 (Tray-Icon & Kontextmenü) gefüllt.

Hotkeys

Default-Hotkeys

TastenkombinationAktion
Ctrl+Shift+1Bereich aufnehmen
Ctrl+Shift+2Aktives Fenster
Ctrl+Shift+3Monitor unter Maus
Ctrl+Shift+4Alle Monitore
Ctrl+Shift+0Tray-Menü anzeigen
Hotkeys sind im Konfigurationsdialog anpassbar. Bei Konflikten zeigt LucentScreen einen Vorschlag für eine freie Kombination.

Capture-Modi

Bereich

Ein Vollbild-Overlay erscheint über allen Monitoren. Mit gedrückter Maustaste ziehen Sie ein Rechteck. ESC bricht ab.

Aktives Fenster

Das aktuell aktive Fenster wird inklusive Frame, aber ohne Schlagschatten, aufgenommen.

Monitor unter Maus

Der Monitor, auf dem sich der Cursor befindet, wird vollständig aufgenommen.

Alle Monitore

Der virtuelle Gesamtbildschirm aller verbundenen Monitore.

Editor

Wird mit AP 6 (Editor) gefüllt.

Verlauf

Wird mit AP 7 (Verlauf) gefüllt.

Konfiguration

Die Konfigurationsdatei liegt unter %APPDATA%\LucentScreen\config.json und wird beim ersten Start aus config.default.json erzeugt.

Vollständiges Schema wird mit AP 1 (Konfiguration) dokumentiert.

Architektur

LucentScreen ist in drei Layer aufgebaut:

  • core — Domain-Logik, GDI+, P/Invoke. Keine WPF-Abhängigkeit.
  • ui — XAML, WPF-Fenster, NotifyIcon, HwndSource-Hotkey-Hook.
  • entry — LucentScreen.ps1: STA-Check, Single-Instance, DPI-Awareness, Application-Loop.

Details: docs/Architektur.md im Quellrepository.

Troubleshooting

Tray-Icon bleibt nach Beenden sichtbar: Windows-Glitch. Über das Icon mit der Maus fahren, dann verschwindet es.
Hotkey funktioniert nicht: Andere Anwendung hat die Kombination bereits registriert. In der Konfiguration auf eine alternative Kombination wechseln.
Screenshot auf zweitem Monitor ist verschoben: DPI-Skalierungs-Mismatch — Windows-Display-Einstellungen prüfen, evtl. App neu starten.
