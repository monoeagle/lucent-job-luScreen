## Was in dieser Session passiert ist

Vier User-Wünsche → ein Patch-Release.

### Wünsche

1. Löschen-Button im Verlauf an die rechte Seite mit eigenem Trenner — gegen versehentliches Anklicken beim Greifen nach Anzeigen/Bearbeiten.
2. Dialoge sollen nur einmal öffnen — vorher legten zweite Tray-Klicks weitere Instanzen oben drauf.
3. Tray-Icon farblich invertieren — schwarz auf dunkler Taskleiste war kaum lesbar.
4. Schloss-Icon pro Verlaufs-Bild gegen Löschen, default offen, Dateisystem-Delete bleibt unangetastet.

### Umsetzung

- **Toolbar-Reordering** (`src/views/history-window.xaml`): `BtnDelete` an die Reihe-Ende geschoben + Border-Trenner davor. Reihenfolge jetzt: Anzeigen / Bearbeiten / Umbenennen / Speicherort / Clipboard / Liste / Refresh · `|` · Druck · `|` · Löschen.
- **Single-Instance-Guard** in `Show-HistoryWindow`, `Show-ConfigDialog`, `Show-AboutDialog`: `$script:OpenWindow`-Slot, Topmost-Toggle + `Activate` holt das offene Fenster nach vorn statt eine zweite Instanz im verschachtelten Dispatcher zu öffnen. **Trap:** ohne explizite `$script:OpenWindow = $null`-Init auf Modul-Ebene wirft `Set-StrictMode -Version Latest` beim ersten Lese-Zugriff `Variable nicht festgelegt`. Initialisierung direkt nach `Set-StrictMode` in allen drei Modulen.
- **Tray-Icon-Inversion** (`src/ui/tray.psm1::_New-TrayIcon`): neuer `-Invert`-Pfad lädt ICO/PNG zu `Bitmap`, rendert mit `ColorMatrix` (Matrix00/11/22 = -1, Matrix40/41/42 = 1) auf neue Bitmap → `GetHicon` → `Icon.FromHandle`. Alpha-Diagonale bleibt 1, deshalb transparent bleibt transparent. `Initialize-Tray -InvertIcon` (Default `$true`) macht den Schalter offiziell.
- **Schloss pro Bild**: `LucentScreen.HistoryEntry`-C#-Klasse um `IsLocked` (mit `INotifyPropertyChanged`) erweitert; `New-HistoryItem` in `src/core/history.psm1` exportiert jetzt `IsReadOnly`; `$mkEntry` setzt `IsLocked = $Item.IsReadOnly`. XAML-`DataTemplate`: Grid mit Thumbnail + `Button x:Name="BtnLock"` rechts oben. Style `LockButton` mit `DataTrigger` auf `IsLocked` schaltet Glyph (E785 offen / E72E zu) + Farben (grün `#15803D` auf `#CCDCFCE7` / rot `#B91C1C` auf `#CCFEE2E2`, Alpha CC ≈ 80 % damit Bild durchscheint). Click-Routing: `LstItems.AddHandler(Button.ClickEvent, ...)` — der gebubblede RoutedEvent vermeidet Subklick-Konflikte mit `ListBoxItem.IsSelected`. `cmdDelete` filtert gesperrte Einträge raus und meldet die Skip-Zahl im Confirm-Text. Persistenz = NTFS-`ReadOnly`-Attribut; **keine** Sidecar-Datei, **kein** Schema-Bump.
