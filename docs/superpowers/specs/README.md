# Specs

Feature-Specs. **Was** und **warum**, nicht **wie**.

## Format

```
<kebab-feature>.md
```

## Vorlage

```markdown
# Spec: <Feature>

## Problem
Welches Problem löst dieses Feature?

## User Story
Als <Rolle> möchte ich <Aktion>, um <Ziel>.

## Akzeptanzkriterien
- [ ] …
- [ ] …

## Out of Scope
- …

## Offene Fragen
- …
```

## Verwandt

- `../plans/` — wie das Feature umgesetzt wird
